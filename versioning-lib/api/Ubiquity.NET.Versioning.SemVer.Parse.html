Remarks

The provider is used to specify the sort ordering of Alphanumeric IDs for the version. The default used if provider is null is CaseSensitive SemVer values. However, an input MAY be a valid CSemVer, and if provider is null the result is a CSemVer which REQUIRES CaseInsensitive. If the source of the string requires case sensitive comparison, then callers should explicitly provide CaseSensitive to exclude the CSemVer and CSemVer-CI variants from consideration (They both include constraints that they use case insensitive comparisons). That is, unless explicitly provided the AlphaNumericOrdering value of all parsed versions depends on the type that is parsed. A SemVer is parsed first, then if possible a CSemVer or CSemVerCI is created from that. If provider requires CaseSensitive then ONLY the SemVer is considered as both CSemVer and CSemVerCI REQUIRE case insensitive comparisons.
